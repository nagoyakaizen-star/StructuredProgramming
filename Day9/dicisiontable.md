### 表の読み解きと整理

画像は3入力（`A`, `B`, `C`）と2出力（`d`, `e`）のテーブルになっています。右端にある謎の列（すべて0）は出力項目名がないため、今回は無視して`d`と`e`を処理対象とします。

| 行 | input A | input B | input C | output d | output e | 2進数(A B C) | 10進数インデックス |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | 1 | 1 | 1 | 0 | 0 | 111 | 7 |
| 2 | 1 | 1 | 0 | 1 | 0 | 110 | 6 |
| 3 | 1 | 0 | 1 | 0 | 0 | 101 | 5 |
| 4 | 1 | 0 | 0 | 1 | 1 | 100 | 4 |
| 5 | 0 | 1 | 1 | 0 | 0 | 011 | 3 |
| 6 | 0 | 1 | 0 | 0 | 0 | 010 | 2 |
| 7 | 0 | 0 | 1 | 0 | 0 | 001 | 1 |
| 8 | 0 | 0 | 0 | 0 | 0 | 000 | 0 |

※出力が `1` になる条件だけを抜き出すと以下のようになります。

* **`d` が 1 になる条件:** `(A=1, B=1, C=0)` または `(A=1, B=0, C=0)`
* **`e` が 1 になる条件:** `(A=1, B=0, C=0)`

### 方法1：`if`文（条件分岐）で書く方法

出力が `1` になる特定のパターンだけを判定し、それ以外は `0` にするシンプルな書き方です。

```c
#include <stdio.h>

void process_with_if(int A, int B, int C, int *d, int *e) {
    // 初期値としてすべて0（デフォルト）を設定
    *d = 0;
    *e = 0;

    // 出力が1になる条件を判定
    if (A == 1 && B == 1 && C == 0) {
        *d = 1;
    } else if (A == 1 && B == 0 && C == 0) {
        *d = 1;
        *e = 1;
    }
}

int main() {
    int d, e;
    
    // テスト: A=1, B=0, C=0 の場合 (d=1, e=1 になるはず)
    process_with_if(1, 0, 0, &d, &e);
    printf("[IF文]   A=1, B=0, C=0 -> d=%d, e=%d\n", d, e);

    // テスト: A=1, B=1, C=0 の場合 (d=1, e=0 になるはず)
    process_with_if(1, 1, 0, &d, &e);
    printf("[IF文]   A=1, B=1, C=0 -> d=%d, e=%d\n", d, e);

    return 0;
}

```

### 方法2：配列（ルックアップテーブル）に値を入れる方法

入力 `A`, `B`, `C` をビットに見立てて「0〜7」のインデックス番号（10進数）に変換し、配列から直接出力を引き出すスマートな方法です。

```c
#include <stdio.h>

void process_with_array(int A, int B, int C, int *d, int *e) {
    // インデックス0〜7に対応する出力を配列として定義
    // インデックスは A*4 + B*2 + C*1 で計算
    int table_d[8] = {0, 0, 0, 0, 1, 0, 1, 0}; // インデックス4と6が1
    int table_e[8] = {0, 0, 0, 0, 1, 0, 0, 0}; // インデックス4だけが1

    // 3つの入力を合成して0〜7の配列添字（インデックス）を作る
    int index = (A << 2) | (B << 1) | C; // A*4 + B*2 + C と同じ

    // 配列から値を代入
    *d = table_d[index];
    *e = table_e[index];
}

int main() {
    int d, e;

    // テスト: A=1, B=0, C=0 (インデックス: 4) -> d=1, e=1 
    process_with_array(1, 0, 0, &d, &e);
    printf("[配列法] A=1, B=0, C=0 -> d=%d, e=%d\n", d, e);

    // テスト: A=1, B=1, C=0 (インデックス: 6) -> d=1, e=0
    process_with_array(1, 1, 0, &d, &e);
    printf("[配列法] A=1, B=1, C=0 -> d=%d, e=%d\n", d, e);

    return 0;
}

```

# Q２　Cプログラムの処理時間を測定するため、処理開始時間と処理終了時間の差を求めるプログラムを書いてください。
A2 
C言語で処理時間を測定する場合、標準ライブラリの `<time.h>` に用意されている `clock()` 関数、または `timespec_get()` 関数（C11以降）を使用するのが一般的です。

測定したい精度（秒単位か、ミリ秒・マイクロ秒単位か）に合わせて使い分けられるよう、**2パターンのプログラム**を用意しました。

---

### 方法1：`clock()` 関数を使う（最も手軽 / ミリ秒単位）

CPUがそのプログラムの処理に費やした時間（CPU時間）を測定します。ミリ秒単位での測定に向いています。

```c
#include <stdio.h>
#include <time.h>

// ダミーの重い処理（時間を稼ぐためのループ）
void heavy_processing() {
    long long sum = 0;
    for (long long i = 0; i < 500000000; i++) {
        sum += i;
    }
}

int main() {
    // 1. 処理開始前のクロック数を取得
    clock_t start_time = clock();

    // 測定したい処理を実行
    heavy_processing();

    // 2. 処理終了後のクロック数を取得
    clock_t end_time = clock();

    // 3. 差分を計算し、秒単位に変換（CLOCKS_PER_SEC で割る）
    double duration = (double)(end_time - start_time) / CLOCKS_PER_SEC;

    printf("処理時間: %f 秒\n", duration);

    return 0;
}

```

---

### 方法2：`timespec_get()` を使う（高精度 / ナノ秒・マイクロ秒単位）

C11規格以降で利用できる、より高精度なシステム時間を測定する方法です。処理が非常に一瞬で終わる場合や、マイクロ秒・ナノ秒単位の精度が欲しい場合はこちらを使用します。

```c
#include <stdio.h>
#include <time.h>

void short_processing() {
    long long sum = 0;
    for (int i = 0; i < 10000000; i++) {
        sum += i;
    }
}

int main() {
    struct timespec start, end;

    // 1. 開始時間を取得
    timespec_get(&start, TIME_UTC);

    // 測定したい処理を実行
    short_processing();

    // 2. 終了時間を取得
    timespec_get(&end, TIME_UTC);

    // 3. 秒とナノ秒それぞれの差分を計算
    long diff_sec = end.tv_sec - start.tv_sec;
    long diff_nsec = end.tv_nsec - start.tv_nsec;

    // ナノ秒の引き算で繰り下がりが発生した場合の補正
    if (diff_nsec < 0) {
        diff_sec -= 1;
        diff_nsec += 1000000000;
    }

    // 秒単位にまとめて小数で表示
    double total_time = diff_sec + (double)diff_nsec / 1000000000;

    printf("処理時間: %ld 秒 と %ld ナノ秒\n", diff_sec, diff_nsec);
    printf("処理時間（小数表示）: %.9f 秒\n", total_time);

    return 0;
}

```

### 注意点

* **`clock()` の限界:** 非常に高速な処理（数マイクロ秒で終わる関数など）を `clock()` で測ろうとすると、差分が `0` になってしまうことがあります。その場合は方法2を選ぶか、測定対象の関数をループで1万回ほど回して、かかった総時間をループ回数で割るアプローチを取ってください。
* **最適化:** コンパイル時に最適化オプション（`-O2` や `-O3` など）をつけると、中身のない空のループ処理などはコンパイラによって「不要な処理」とみなされ、消去されて測定時間が0秒になることがあります。本番のコードを測定する際は問題ありませんが、テスト用のダミー処理を走らせる際はお気をつけください。



* **`if`文のメリット:** 条件が少ないときに見やすく、直感的にコードが理解しやすいです。
* **配列法のメリット:** 条件（行数）がどれだけ増えても、`if`文のように複雑にネストせず、処理速度が常に一定（高速）になります。組込み開発などでディシジョンテーブルを実装する際は、この配列を使ったルックアップテーブル方式がよく好まれます。
