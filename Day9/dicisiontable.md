アップロードしていただいた真理値表（ディシジョンテーブル）から、入力と出力の関係を整理した上で、C言語のプログラムを2通り（`if`文を使う方法、配列のルックアップテーブルを使う方法）で作成します。

### 表の読み解きと整理

画像は3入力（`A`, `B`, `C`）と2出力（`d`, `e`）のテーブルになっています。右端にある謎の列（すべて0）は出力項目名がないため、今回は無視して`d`と`e`を処理対象とします。

| 行 | input A | input B | input C | output d | output e | 2進数(A B C) | 10進数インデックス |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | 1 | 1 | 1 | 0 | 0 | 111 | 7 |
| 2 | 1 | 1 | 0 | 1 | 0 | 110 | 6 |
| 3 | 1 | 0 | 1 | 0 | 0 | 101 | 5 |
| 4 | 1 | 0 | 0 | 1 | 1 | 100 | 4 |
| 5 | 0 | 1 | 1 | 0 | 0 | 011 | 3 |
| 6 | 0 | 1 | 0 | 0 | 0 | 010 | 2 |
| 7 | 0 | 0 | 1 | 0 | 0 | 001 | 1 |
| 8 | 0 | 0 | 0 | 0 | 0 | 000 | 0 |

※出力が `1` になる条件だけを抜き出すと以下のようになります。

* **`d` が 1 になる条件:** `(A=1, B=1, C=0)` または `(A=1, B=0, C=0)`
* **`e` が 1 になる条件:** `(A=1, B=0, C=0)`

---

### 方法1：`if`文（条件分岐）で書く方法

出力が `1` になる特定のパターンだけを判定し、それ以外は `0` にするシンプルな書き方です。

```c
#include <stdio.h>

void process_with_if(int A, int B, int C, int *d, int *e) {
    // 初期値としてすべて0（デフォルト）を設定
    *d = 0;
    *e = 0;

    // 出力が1になる条件を判定
    if (A == 1 && B == 1 && C == 0) {
        *d = 1;
    } else if (A == 1 && B == 0 && C == 0) {
        *d = 1;
        *e = 1;
    }
}

int main() {
    int d, e;
    
    // テスト: A=1, B=0, C=0 の場合 (d=1, e=1 になるはず)
    process_with_if(1, 0, 0, &d, &e);
    printf("[IF文]   A=1, B=0, C=0 -> d=%d, e=%d\n", d, e);

    // テスト: A=1, B=1, C=0 の場合 (d=1, e=0 になるはず)
    process_with_if(1, 1, 0, &d, &e);
    printf("[IF文]   A=1, B=1, C=0 -> d=%d, e=%d\n", d, e);

    return 0;
}

```

### 方法2：配列（ルックアップテーブル）に値を入れる方法

入力 `A`, `B`, `C` をビットに見立てて「0〜7」のインデックス番号（10進数）に変換し、配列から直接出力を引き出すスマートな方法です。

```c
#include <stdio.h>

void process_with_array(int A, int B, int C, int *d, int *e) {
    // インデックス0〜7に対応する出力を配列として定義
    // インデックスは A*4 + B*2 + C*1 で計算
    int table_d[8] = {0, 0, 0, 0, 1, 0, 1, 0}; // インデックス4と6が1
    int table_e[8] = {0, 0, 0, 0, 1, 0, 0, 0}; // インデックス4だけが1

    // 3つの入力を合成して0〜7の配列添字（インデックス）を作る
    int index = (A << 2) | (B << 1) | C; // A*4 + B*2 + C と同じ

    // 配列から値を代入
    *d = table_d[index];
    *e = table_e[index];
}

int main() {
    int d, e;

    // テスト: A=1, B=0, C=0 (インデックス: 4) -> d=1, e=1 
    process_with_array(1, 0, 0, &d, &e);
    printf("[配列法] A=1, B=0, C=0 -> d=%d, e=%d\n", d, e);

    // テスト: A=1, B=1, C=0 (インデックス: 6) -> d=1, e=0
    process_with_array(1, 1, 0, &d, &e);
    printf("[配列法] A=1, B=1, C=0 -> d=%d, e=%d\n", d, e);

    return 0;
}

```

### どちらを選ぶべき？

* **`if`文のメリット:** 条件が少ないときに見やすく、直感的にコードが理解しやすいです。
* **配列法のメリット:** 条件（行数）がどれだけ増えても、`if`文のように複雑にネストせず、処理速度が常に一定（高速）になります。組込み開発などでディシジョンテーブルを実装する際は、この配列を使ったルックアップテーブル方式がよく好まれます。
