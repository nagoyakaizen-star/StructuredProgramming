## Answer 1
GitHubで管理しているファイルを「隣のフォルダ（同じ親フォルダを持つ別のフォルダ）」に移動させる方法を、CUI（Gitコマンド）**と**GUI（Webブラウザ・アプリ）の2通りで解説します。

---

## 前提のフォルダ構成（例）

以下のような構成で、`folder-A` にある `document.md` を、隣の `folder-B` に移動させると仮定します。

```text
repository-root/
├── folder-A/
│   └── document.md  <-- これを移動したい
└── folder-B/        <-- ここに移動させたい

```

---

## 1. CUI（コマンドライン）で行う方法

コマンドラインで行う場合は、単純にファイルを動かすだけでなく、「Gitに移動したことを追跡させる（履歴を維持する）」ために `git mv` コマンドを使用するのがベストプラクティスです。

1. **リポジトリのディレクトリに移動する:** ターミナル/コマンドプロンプト.
まず、ローカルのリポジトリがあるフォルダに移動します。

```bash
cd /path/to/your/repository-root

```


2. **git mv コマンドでファイルを移動する:** Gitへの移動指示.
`git mv [移動元] [移動先]` の形式でコマンドを実行します。

```bash
git mv folder-A/document.md folder-B/

```

これでファイルが物理的に移動し、Git側でも「名前変更（Rename）/移動」として自動的にインデックス（ステージングエリア）に登録されます。


3. **変更をコミットする:** 履歴の記録.
移動を確定させるためのコミットメッセージを書いてコミットします。

```bash
git commit -m "Move document.md from folder-A to folder-B"

```


4. **GitHubに反映する:** プッシュ.
最後に、ローカルの変更をリモートリポジトリ（GitHub）に送信します。

```bash
git push origin main

```


---

## 2. GUIで行う方法

ローカルにクローン（ダウンロード）せず、**GitHubのWebブラウザ画面上で直接移動させる**方法です。

実は、GitHubのWeb画面には直感的な「ドラッグ＆ドロップでのファイル移動」機能はありませんが、**ファイル名のパスを直接書き換える**ことでフォルダ移動が可能です。

1. **ファイル編集画面を開く:** GitHub（ブラウザ）.
GitHubで該当のファイル（`document.md`）を開き、右上の **鉛筆アイコン（Edit this file）** をクリックして編集モードに入ります。


2. **ファイル名のパスを書き換える:** ここがポイント.
画面上部にあるファイル名が表示されている入力ボックス（`folder-A/document.md` となっている部分）をクリックします。
キーボードの `Backspace`（Macは `Delete`）キーを押し、**`folder-A/` を削除して `../folder-B/` または新しいパス（`folder-B/`）に書き換えます。**

> **Tip:** パス入力欄で `../` と入力すると、一つ上の親フォルダに戻ることができます。


3. **変更をコミットする:** 保存と反映.
パスを書き換えたら、右上の **「Commit changes...」** ボタンをクリックします。
コミットメッセージを入力し、直接 `main` ブランチにコミットするか、プルリクエストを作成して保存します。これで移動完了です。


*※「GitHub Desktop」アプリを使用している場合は、パソコンのエクスプローラーやFinderで通常通りファイルを `folder-B` にドラッグ＆ドロップで移動させた後、アプリを開くと自動的に「移動（Renamed）」として検出されるので、そのままコミット・プッシュすればOKです。*
